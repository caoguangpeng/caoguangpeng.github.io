使用Github Pages和Hexo搭建博客

发表于 2015-07-13   |   分类于 其他   |  

作为一名软件开发者，有个自己的博客还是非常有必要的，此次搭建博客，选了Github Pages和Hexo，下面分别介绍下：

Github Pages

Github Pages可以被认为是用户编写的、托管在github上的静态网页。有300M免费空间，省去了自己购买空间的钱，是搭建博客的不二之选。
要想拥有自己的一个空间，步骤如下：

注册Github

注册很简单，上官网，注册你的用户名和邮箱

配置Github

这块主要是指上传ssh公钥，github支持ssh协议，要想上传本地的文件至github，必须有密钥配对。

检查是否有ssh key

我用的是MacBook，下面所有的操作都是在这个环境下的。
打开terminal,

$ cd ~/.ssh

如果提示：No such file or directory，说明没有，继续步骤2.2；否则就跳过2.2。

生成ssh key

$ ssh-keygen -t rsa -C 你的邮箱
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

然后系统会要你输入密码：

Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>

这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。

之后敲回车，再查看是否生成默认名字是id_rsa(私钥)和id_rsa.pub(公钥)两个文件，如果都有，那就成功了。

上传ssh key(.pub后缀)到github

打开github网站，进入用户Settings—->SSH keys —-> Add SSH key，添加id_rsa.pub的内容即可。

测试ssh key是否设置成功

$ ssh -T git@github.com

如果出现例如

Hi caoguangpeng! You've successfully authenticated, but GitHub does not provide shell access.

那就设置成功了。

创建GitHub Pages仓库

和github建立连接后，就可以很方便地使用GitHub Pages服务了，只要在当前账户下建立形如username.github.io的仓库即可，创建过程都默认设置好了。

Hexo

Hexo是一个基于Node.js的快速、简洁且高效的博客框架，支持Markdown格式，可以很方便地生成静态网页，并托管在github上，和Github Pages配合使用，顶好！

安装Hexo

sudo npm install -g hexo    (-g表明全局安装，不加的话就只能在当前目录下安装)

安装过程如果很慢或者出现类似以下的错误：

{ [Error: Cannot find module './build/Release/DTraceProviderBindings'] code: 'MODULE_NOT_FOUND' }
{ [Error: Cannot find module './build/default/DTraceProviderBindings'] code: 'MODULE_NOT_FOUND' }
{ [Error: Cannot find module './build/Debug/DTraceProviderBindings'] code: 'MODULE_NOT_FOUND' }

那就试试：

sudo npm install -g hexo --no-optional

初始化目标文件夹

方法1：

先创建个文件夹，比如blog，然后在blog目录下，输入 hexo init

方法2：

直接输入 hexo init blog

生成文件

在第2步成功后，会提示先运行“npm install”，建议输入：

npm install --no-optional

之后运行：

hexo generate(或者缩写g,以下都用缩写了)

查看效果：

hexo server(或者缩写s)

然后在打开浏览器 localhost:4000 ,就可以看到,本地已经安装好了,默认会有一篇Hello World的文章。

note:Hexo3.0以上，把client和server分离了，可能运行hexo s时会出错，那么就单独再安装server模块即可

部署

还记得一开始的github仓库吗？之前的操作都在本地进行，这步操作我们将把我们写好的文章部署到github上，步骤如下：

修改站点_config.yml

在你刚才创建的blog根目录下，会有个_config.yml，编辑它，在最底下修改为

deploy:
    type: github      
    repository:git@github.com:caoguangpeng/caoguangpeng.github.io.git //当然你要改成自己的用户名   
    branch: master

执行部署

hexo g
hexo deploy(缩写d)

执行完以后,如果报错 Error: Deployer not found : github,则执行如下命令:

npm install hexo-deployer-git --save

同时再修改 _config.yml :

deploy:
    type: git

然后再执行：

hexo g
hexo d

最后访问:http://caoguangpeng.github.io/ (用户名改成自己的),就可以看到了.

注意,每次修改本地目录下的文件后，需要hexo generate才能保存;另外每次deploy，都是覆盖操作，这样CNAME文件就消失了，为了避免每次都创建，我们可以在source目录下，创建CNAME，然后每次部署后，就自动会上传CNAME的

好了，至此博客搭建的过程就大功告成了，是不是很简单，赶紧动手试试吧！后续我会讲讲如何修改hexo的主题，以及如何写博客，敬请期待吧…
